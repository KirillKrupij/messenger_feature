# Фича чата (Messenger)

## 📌 Обзор

Это **рабочая фича мессенджера**, извлечённая из production-проекта. Фича публикуется в качестве демонстрации архитектурных решений, качества кода и подхода к тестированию.

Фича представляет собой полнофункциональный мессенджер с поддержкой:
- Личных и групповых чатов
- Реального времени через WebSocket
- Вложений файлов, изображений, видео
- Реакций на сообщения
- Индикаторов набора текста
- Статусов онлайн/офлайн
- Поиска по чатам и сообщениям
- Пагинации и подгрузки истории

## 🏗️ Архитектура

Фича построена по принципам **Clean Architecture** с чётким разделением на слои:

### 1. Domain (Бизнес-логика)
- **Entities**: `Chat`, `Message`, `UserStatus`
- **Repositories**: Абстрактные интерфейсы (`ChatRepository`)
- **Use Cases**: Бизнес-сценарии (`GetTotalUnreadCountUseCase`)

### 2. Data (Доступ к данным)
- **Models**: DTO-модели для API (`ChatModel`, `MessageModel`)
- **Repositories**: Конкретные реализации (`ChatRepositoryImpl`)
- **Data Sources**: `ChatRemoteDataSource` (REST API + WebSocket)

### 3. Presentation (UI)
- **State Management**: Riverpod (`StateNotifierProvider`)
- **Widgets**: Полностью реактивные виджеты
- **Providers**: `ChatNotifier`, `AuthNotifier`

## 🚀 Ключевые особенности

### Реальное время
- WebSocket-подключение с автоматическим переподключением
- Мгновенная доставка сообщений
- Индикаторы набора текста в реальном времени
- Обновление статусов пользователей

### Расширенный UI
- Адаптивный дизайн под мобильные и десктоп
- Кастомные анимации сообщений
- Поддержка жестов (свайпы, долгое нажатие)
- Темная/светлая тема

### Безопасность и надёжность
- JWT-авторизация
- Валидация входящих данных
- Обработка ошибок с пользовательскими сообщениями
- Оптимистичные обновления UI

## 🧪 Тестирование

Фича включает **полноценный набор тестов**, демонстрирующий профессиональный подход к quality assurance:

### Unit-тесты
- **Use Cases**: `GetTotalUnreadCountUseCase` (успех, ServerFailure, CacheFailure)
- **Providers**: `ChatNotifier` (загрузка чатов, отправка сообщений, выделение)
- **Моки**: Используется `mocktail` для изолированного тестирования

### Widget-тесты
- **MessengerWidget**: Проверка отображения и базовых взаимодействий
- **Покрытие**: 2 из 4 тестов проходят, остальные требуют дополнительной настройки моков

### Интеграция
- Все unit-тесты успешно проходят (10/10)
- Настроена инфраструктура для расширения тестирования

## 🛠️ Технологический стек

- **Flutter 3.0+** с null safety
- **Riverpod 2.4** для state management
- **Dartz** для функционального error handling
- **WebSocket** для реального времени
- **Mocktail** для тестирования

## 🔧 Как запустить

Фича интегрирована в основной проект DEMS. Для изолированного запуска потребуется:

1. Настроить зависимости в `pubspec.yaml`
2. Поднять бэкенд-сервер с WebSocket-поддержкой
3. Настроить `.env` с API-ключами
